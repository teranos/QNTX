# Release Notes — March 29 to April 10, 2026

Nebula gained a GPU pick buffer that uses a Metal R32Uint texture with depth testing to identify the nearest particle at any pixel, letting users hover and click individual tokens in the probability cloud. Examine mode isolates a single keyframe's probability distribution, dims surrounding text by sentence proximity, and flies the camera to the distribution center. Keyboard navigation arrived for both canvas and nebula — hjkl moves spatially across glyphs, bracket keys step through keyframes, and Cmd+Enter executes compositions. Canvas open time dropped from 2282ms to 588ms by making WASM init, backend canvas sync, and plugin glyph loading non-blocking, with the canvas rendering immediately from local IndexedDB state.

The @qntx/glyphs SDK reached 0.4.0 across two releases: 0.3.0 extracted wireExpandToWindow and canvas-window into the package while renaming response-glyph to result-glyph, and 0.4.0 fixed window content wrapping by creating a wrapper div instead of mutating the user's element. Thirteen re-export shim files were deleted as all imports now point to @qntx/glyphs directly. A temporal dead zone bug where an element was used before declaration in result-glyph was caught and fixed during the extraction. The response glyph no longer jumps downward on click — the root cause was position:relative overwriting position:absolute, which broke drag coordinate math via cumulative offsetTop drift.

TAO gained vision support through mtmd, auto-detecting mmproj GGUF files in the model directory and routing image attachments through the CLIP encoder, verified end-to-end with Qwen2.5-VL 3B. The core LLM routing layer now enforces a priority semaphore and sliding-window rate limiter before any request reaches a provider, with queue depth capped at 20 and RESOURCE_EXHAUSTED returned when full. Progressive backpressure rejects low-priority requests earlier as the queue fills, and a configurable cooldown pause between inference runs prevents thermal saturation. A crash in scry where prompts exceeding llama.cpp's default 512-token n_batch triggered a fatal GGML_ASSERT was fixed by setting n_batch equal to n_ctx.

EmbeddingService was exposed over gRPC so plugins can request text-to-vector embeddings without direct ONNX runtime access — Werf validated this by clustering 521 job descriptions into 369 roles via FAISS. Weave attestation creation moved from scry into the core LLM routing layer, giving every provider (scry, openrouter, future) observability for free without writing weaves directly. Weave counts are now wired into the Pulse ticker's periodic summary. Per-plugin named loggers replaced the generic plugin-loader label, and keepalive logs now include both plugin name and version.

StreamChat was wired through LLMServer to provider plugins, completing the unified inference path. The root Makefile's install target was removed since each plugin owns its own build. The deprecated Ax CLI configuration was stripped from am.example.toml, and hardcoded ports were removed from CLAUDE.md in favor of am.toml as the single source. A documentation sweep added or revised READMEs for typegen, server, sitegen, and several internal packages, and recorded TODO codes for known limitations across PIK, NDOC, WLOG, and CWEV.
